## Phase 1: 페이지 정리 및 네비게이션 단순화

### 개요
프로젝트를 단순화하기 위해 불필요한 페이지와 라우트를 제거하고, 네비게이션을 핵심 기능(홈, 인증, 프로필)만 남깁니다. 이를 통해 사용자가 도시 정보에 집중할 수 있도록 합니다.

### 현재 상태 파악
- 홈페이지(`/`): 유지
- 로그인 페이지(`/login`): 유지
- 회원가입 페이지(`/register`): 유지
- 인증 라우트(`/auth/*`): 유지
- 기타 페이지(cities, ranking, map, community 등): 삭제 필요
- Navigation 컴포넌트: 메뉴 항목 정리 필요

### 수정/개선 사항 체크리스트

- [x] `app/community` 디렉토리 삭제 (존재하는 경우)
- [x] Navigation 컴포넌트에서 메뉴 항목 배열 정리 (도시탐색, 랭킹, 지도, 커뮤니티 제거)
- [x] Navigation 컴포넌트는 로고, 로그인/로그아웃, 프로필 버튼만 유지
- [x] MobileMenu 컴포넌트도 동일하게 정리
- [x] 삭제된 페이지로의 모든 Link 제거 확인

## 참고 사항

### 기술 스택
- Next.js 15 (App Router)
- React 19 RC
- TypeScript
- Tailwind CSS
- Supabase Auth
- Skeuomorphic 디자인 시스템

### 데이터베이스
현재 단계에서는 데이터베이스를 사용하지 않고 Mock 데이터(`lib/mock-data.ts`)로 진행합니다.

### 디자인 시스템
모든 컴포넌트는 Skeuomorphic 디자인을 따릅니다:
- `.card-skeu` - 3D 카드
- `.btn-skeu` - 3D 버튼
- `.input-skeu` - 3D 입력 필드
- `.tag-skeu` - 3D 태그/배지
