## 1. 개요 (Overview)

### 1.1 프로젝트 배경
디지털 노마드 라이프스타일이 전 세계적으로 확산되고 있으나, 한국 내에서 디지털 노마드로 생활하고자 하는 사람들을 위한 통합 정보 플랫폼이 부재한 상황입니다. 기존 서비스들은 글로벌 도시를 중심으로 구성되어 있어 한국 특화 정보(카페 문화, 빠른 인터넷, 합리적인 생활비, 대중교통 등)가 부족합니다.

### 1.2 제품 비전
한국에서 디지털 노마드로 살고 싶은 사람들이 **가장 먼저 찾는 플랫폼**이 되어, 도시별 생활 조건을 한눈에 비교하고 자신에게 맞는 도시를 쉽게 찾을 수 있도록 돕습니다.

## 2. 문제 정의 (Problem Statement)

### 2.1 현재 상황
- **정보 분산**: 도시별 생활비, 카페 정보, 인터넷 속도, 주거비 등이 네이버 블로그, 유튜브, 개인 SNS 등에 산재
- **신뢰성 부족**: 객관적인 데이터 부재, 개인 의견에 따라 정보 편차 큼
- **비교 불가**: 여러 도시를 동시에 비교할 수 있는 도구 없음
- **한국 특화 정보 부재**: 글로벌 노마드 플랫폼은 한국 문화(카페 코워킹, 찜질방, 고시원 등)에 대한 이해 부족

### 2.2 핵심 문제
> "한국에서 디지털 노마드로 생활하고 싶은 사람들이 **어느 도시에서 살지 결정하기 위해 필요한 정보를 한 곳에서 쉽게 찾을 수 없다.**"

### 2.3 영향
- 잠재 사용자들이 정보 수집에 과도한 시간 소비 (평균 20-30시간)
- 부정확한 정보로 인한 잘못된 도시 선택 → 이동 비용 낭비
- 노마드 커뮤니티 형성 어려움 → 고립된 생활

## 3. 타겟 사용자 (Target Users)

### 3.1 주요 타겟 (Primary)

**페르소나 1: 프리랜서 개발자 "김코딩" (28세, 남성)**
- **직업**: 프론트엔드 개발자 (프리랜서)
- **수입**: 월 400-500만원
- **특징**:
  - 서울 월세 부담으로 지방 이주 고려
  - 빠른 인터넷과 조용한 카페 환경 필수
  - 주말엔 취미 생활(등산, 카페 투어)
  - 노트북 하나로 어디서든 일 가능
- **니즈**: "월세 100만원 이하면서 카페가 많고 인터넷 빠른 도시 찾기"

**페르소나 2: 콘텐츠 크리에이터 "박유튜버" (25세, 여성)**
- **직업**: 유튜버/블로거
- **수입**: 월 300-400만원 (불규칙)
- **특징**:
  - SNS 활발, 인스타그램 팔로워 5만명
  - 콘텐츠 소재가 풍부한 도시 선호
  - 감성적인 공간, 포토스팟 중요
  - 다른 크리에이터와의 네트워킹 중시
- **니즈**: "감성 있고 콘텐츠 찍기 좋으면서 생활비 저렴한 도시"

**페르소나 3: 원격 근무자 "이리모트" (32세, 남성)**
- **직업**: 마케터 (풀타임 원격 근무)
- **수입**: 월 500-600만원 (고정)
- **특징**:
  - 회사 정책으로 거주지 자유
  - 오전 10시-오후 7시 근무 (화상회의 많음)
  - 워라밸 중시, 퇴근 후 여가 활동
  - 장기 거주 (3개월 이상) 고려
- **니즈**: "안정적인 인터넷, 조용한 주거 환경, 문화생활 가능한 도시"

### 3.2 공통 특성
- **연령**: 25-35세 (밀레니얼 세대)
- **기술 능숙도**: 높음 (PC/모바일 자유자재)
- **SNS 활용**: 인스타그램, 유튜브, 블로그 등에서 정보 소비 및 생산
- **직업**: 개발자, 디자이너, 작가, 마케터, 콘텐츠 크리에이터 등
- **가치관**: 장소 독립성, 경험 중시, 워라밸, 자기계발
- **행동 패턴**:
  - 의사결정 전 철저한 리서치
  - 후기/리뷰 신뢰
  - 커뮤니티 참여 적극적
  - 새로운 도시 탐험 즐김

### 3.3 부 타겟 (Secondary)
- 은퇴 후 지방 이주 고려하는 50-60대 (원격 컨설팅)
- 대학생 창업가 (학교 근처 벗어나 저렴한 도시에서 스타트업 운영)
- 주말 노마드 (평일 서울 근무, 주말 지방 체류)

## 4. 목표 및 성공 지표 (Goals & Success Metrics)

### 4.1 비즈니스 목표

**단기 목표 (3개월)**
1. 홈페이지 런칭 및 초기 사용자 확보
2. 15-20개 주요 도시 데이터 구축
3. 커뮤니티 형성 시작 (밋업 기능)

**중기 목표 (6개월)**
1. 월간 활성 사용자(MAU) 10,000명 달성
2. 유료 멤버십 전환율 5% 달성
3. 전국 50개 도시 커버

**장기 목표 (1년)**
1. 한국 디지털 노마드 플랫폼 1위 달성
2. 월간 활성 사용자(MAU) 50,000명
3. 유료 회원 2,500명 (MRR 2,500만원)

### 4.2 제품 목표 (홈페이지 한정)

| 번호 | 목표 | 설명 |
|------|------|------|
| 1 | 정보 접근성 향상 | 사용자가 3클릭 이내에 원하는 도시 정보에 도달 |
| 2 | 직관적 비교 | 도시 카드 UI로 한눈에 여러 도시 비교 가능 |
| 3 | 신뢰도 구축 | 실제 사용자 리뷰와 객관적 데이터 제공 |

## 5. 제안 해결책 (Proposed Solution)

### 5.1 핵심 컨셉
**"한눈에 비교하고, 바로 평가하고, 함께 만나는 플랫폼"**

1. **한눈에 비교**: 도시 카드 UI로 핵심 정보 시각화
2. **바로 평가**: 별점/리뷰 시스템으로 사용자 참여 유도
3. **함께 만나**: 밋업 기능으로 온/오프라인 커뮤니티 통합

### 5.2 차별화 포인트

| 기존 서비스 | 우리 서비스 |
|------------|------------|
| 글로벌 도시 중심 | **한국 도시 특화** |
| 영어 위주 | **한국어, 한국 문화** |
| 숙박 중심 (호텔/에어비앤비) | **월세, 고시원, 게스트하우스 포함** |
| 코워킹 스페이스 | **카페 코워킹 문화 반영** |
| 온라인 커뮤니티 | **온/오프라인 밋업 통합** |
| 정적 데이터 | **실시간 날씨, 미세먼지, 사용자 리뷰** |

### 5.3 핵심 기능 (홈페이지)

**MVP 기능 (필수)**
1. 도시 카드 리스트 (15-20개 도시)
2. 필터링 & 정렬
3. 실시간 통계 (리뷰 수, 밋업 수, 회원 수)
4. 밋업 섹션 (다가오는 4개 밋업)
5. 최근 리뷰 섹션 (3개)
6. 도시 추천 테스트 CTA
7. 뉴스레터 구독

**Nice-to-have 기능 (선택)**
- 지역별 인기 도시 섹션
- 도시별 날씨/미세먼지 실시간 표시
- 추천 카페 미리보기

## 6. 기능 요구사항 (Functional Requirements)

### 6.1 네비게이션 바

#### FR-001: 로고 및 메인 메뉴
- **설명**: 상단 고정 네비게이션 바
- **요구사항**:
  - 로고 클릭 시 홈페이지로 이동
  - 메뉴: [🏙️ 도시탐색] [📊 랭킹] [🗺️ 지도] [👥 커뮤니티]
  - 우측: [🔍 검색] [로그인] 버튼
  - 스크롤 시 상단 고정 (sticky)
  - 모바일: 햄버거 메뉴로 전환
- **우선순위**: P0 (Must Have)

### 6.2 히어로 섹션 (Hero Section)

#### FR-002: 헤드라인 및 CTA
- **설명**: 메인 헤드라인과 핵심 통계, 주요 CTA
- **요구사항**:
  - 헤드라인: "대한민국 디지털 노마드를 위한 최고의 도시 찾기"
  - 통계: "127개 도시 | 3,482개 리뷰 | 월 25회 밋업"
  - CTA 버튼: "나에게 맞는 도시 찾기 →"
  - 버튼 클릭 시 도시 추천 테스트 페이지로 이동
  - 배경: 그라데이션 또는 한국 도시 이미지
- **우선순위**: P0 (Must Have)

### 6.3 검색 및 필터 바

#### FR-003: 통합 검색 바
- **설명**: 도시 검색 및 필터링 인터페이스
- **요구사항**:
  - 검색창: 도시명 자동완성 (예: "서울", "부산", "제주")
  - 필터:
    - 예산 슬라이더 (50만원 - 300만원)
    - 정렬 드롭다운: [종합점수순 ▼] [가성비순] [리뷰많은순] [신규도시순]
  - 태그 필터: [📍전체지역] [💰생활비] [☕카페] [🏠주거] [📶인터넷] [🎨여가]
  - 선택 시 즉시 하단 카드 필터링 (AJAX)
- **우선순위**: P0 (Must Have)

### 6.4 도시 카드 리스트

#### FR-004: 도시 카드 컴포넌트
- **설명**: 각 도시 정보를 카드 형태로 표시
- **카드 구성**:
  ```
  ┌─────────────────────────┐
  │  [도시 대표 이미지]      │
  │                         │
  │  📍 도시명 (지역)        │
  │  ⭐⭐⭐⭐⭐ 평점 (리뷰수)│
  │                         │
  │  점수 바 (4개 항목)      │
  │  ☕ 카페    ▓▓▓▓▓      │
  │  💰 생활비  ▓▓░░░      │
  │  📶 인터넷  ▓▓▓▓▓      │
  │  🏠 주거비  ▓▓░░░      │
  │                         │
  │  💵 월 평균 생활비       │
  │  🌡️ 현재 날씨           │
  │  💨 미세먼지 지수        │
  │                         │
  │  #태그1 #태그2 #태그3   │
  │                         │
  │  [상세보기] [평가하기]   │
  └─────────────────────────┘
  ```
- **요구사항**:
  - 그리드 레이아웃 (데스크톱: 3열, 태블릿: 2열, 모바일: 1열)
  - 이미지: 16:9 비율, lazy loading
  - 평점: 별 5개 시스템, 소수점 1자리
  - 점수 바: 5점 만점, 색상 그라데이션 (빨강→노랑→초록)
  - 태그: 최대 3개 표시
  - 호버 시: 카드 살짝 상승 애니메이션
  - 카드 클릭 시: 도시 상세 페이지로 이동
- **우선순위**: P0 (Must Have)

#### FR-005: 도시 카드 섹션 구분
- **설명**: 카드를 인기/추천 등으로 섹션 구분
- **요구사항**:
  - "인기 도시" 섹션: 평점 높은 순 상위 3개
  - "추천 도시" 섹션: 다음 3개
  - 각 섹션에 제목 표시
  - 무한 스크롤 또는 "더 보기" 버튼
- **우선순위**: P1 (Should Have)

### 6.5 실시간 통계 섹션

#### FR-006: 통계 대시보드
- **설명**: 플랫폼 활동성을 보여주는 통계
- **요구사항**:
  - 6개 지표 카드:
    - 👥 전체 리뷰 수
    - 🏙️ 등록된 도시 수
    - 🍻 이달의 밋업 수
    - ✈️ 이달의 여행자 수
    - ☕ 추천 카페 수
    - 🆕 신규 회원 (월)
  - 각 카드: 아이콘 + 숫자 + 설명
  - 숫자: 카운트업 애니메이션 (0→실제값)
  - 배경: 연한 회색 또는 그라데이션 박스
- **우선순위**: P1 (Should Have)

### 6.6 지역별 인기 도시 섹션

#### FR-007: 지역별 랭킹
- **설명**: 한국을 6개 권역으로 나눠 각 지역 인기 도시 표시
- **요구사항**:
  - 권역: 서울/경기, 부산/경남, 제주, 강원, 전라, 경상
  - 각 권역별 상위 2-4개 도시
  - 도시명 + 평점 + 리뷰 수
  - 클릭 시 해당 도시 상세 페이지로 이동
  - 레이아웃: 2열 또는 3열 그리드
- **우선순위**: P1 (Should Have)

### 6.7 밋업 섹션

#### FR-008: 다가오는 밋업 리스트
- **설명**: 가까운 날짜 순 밋업 4개 표시
- **카드 구성**:
  ```
  ┌────────────────────────┐
  │ 📍 도시명                │
  ```

### 7.3 인터랙션 & 애니메이션

#### UX-001: 로딩 상태
- 페이지 로드: 스켈레톤 UI (카드 형태)
- 필터 적용: 로딩 스피너 + 희미하게 처리
- 이미지 로드: 블러 placeholder → 선명하게

#### UX-002: 호버 효과
- 도시 카드: 살짝 상승 (translateY: -8px) + 그림자 확대
- 버튼: 배경색 변화 + 크기 약간 확대
- 링크: 밑줄 표시

#### UX-003: 클릭 피드백
- 버튼: 클릭 시 살짝 눌림 효과 (scale: 0.95)
- 카드: 클릭 시 페이지 전환 애니메이션

#### UX-004: 스크롤 효과
- 네비게이션 바: 스크롤 시 배경 불투명도 증가
- 카드: Fade-in 애니메이션 (스크롤 시 순차적으로 나타남)
- "맨 위로" 버튼: 스크롤 500px 이상 시 우측 하단 표시

#### UX-005: 토스트 알림
- 위치: 화면 상단 중앙
- 타입: 성공(초록), 에러(빨강), 정보(파랑)
- 자동 사라짐: 3초 후
- 예시: "뉴스레터 구독이 완료되었습니다!"

### 7.4 접근성 (Accessibility)

#### A11Y-001: 키보드 네비게이션
- Tab 키로 모든 인터랙티브 요소 접근 가능
- Focus 상태 명확하게 표시 (outline)
- Esc 키로 모달/드롭다운 닫기

#### A11Y-002: 스크린 리더
- 모든 이미지에 alt 텍스트
- 버튼/링크에 aria-label
- 랜드마크 영역 (header, nav, main, footer)

#### A11Y-003: 색상 대비
- WCAG 2.1 AA 준수 (최소 4.5:1)
- 색상만으로 정보 전달 금지 (아이콘/텍스트 병행)

## 8. 기술 요구사항 (Technical Requirements)

### 8.1 기술 스택

**Frontend**
- Framework: Next.js 15 (App Router)
- Language: TypeScript
- Styling: Tailwind CSS
- UI Components: shadcn/ui
- State Management: Zustand (경량) 또는 React Context
- Form: React Hook Form + Zod (유효성 검사)
- Animation: Framer Motion
- all ui component must created by skeuomorphism

**Backend (MVP에서는 간소화 가능)**
- API: Next.js API Routes
- Database: PostgreSQL (Supabase)
- ORM: Prisma
- Authentication: NextAuth.js (나중에 추가)

**External APIs**
- 날씨: 기상청 API (https://www.data.go.kr)
- 미세먼지: 한국환경공단 API
- 지도: 카카오맵 API

**Hosting & Deployment**
- Frontend: Vercel
- Database: Supabase (PostgreSQL)
- CDN: Cloudflare
- 이미지: Vercel Image Optimization

### 8.2 성능 요구사항

| 항목 | 목표 | 측정 도구 |
|------|------|----------|
| First Contentful Paint (FCP) | < 1.8초 | Lighthouse |
| Largest Contentful Paint (LCP) | < 2.5초 | Lighthouse |
| Cumulative Layout Shift (CLS) | < 0.1 | Lighthouse |
| Time to Interactive (TTI) | < 3.8초 | Lighthouse |
| Total Blocking Time (TBT) | < 300ms | Lighthouse |
| Lighthouse Score | > 90 | Lighthouse |

**최적화 전략**:
- 이미지: WebP 포맷, lazy loading, responsive images
- 코드: Code splitting, tree shaking
- 폰트: 서브셋 폰트, font-display: swap
- 캐싱: CDN, browser cache, API response cache

## 9. 데이터 모델 (Data Model)

### 9.1 초기 데이터 구조

**City (도시)**
```typescript
interface City {
  id: string;
  name: string;           // "서울 성수동"
  region: string;         // "서울/경기"
  slug: string;           // "seoul-seongsu"
  imageUrl: string;
  description: string;

  // 점수
  overallRating: number;  // 4.8
  cafeRating: number;     // 5.0
  costRating: number;     // 2.5
  internetRating: number; // 5.0
  housingRating: number;  // 2.8

  // 통계
  reviewCount: number;    // 327
  likeCount: number;

  // 생활비
  monthlyCost: number;    // 1850000 (원)
  rentCost: number;       // 월세
  cafeCost: number;       // 카페 아메리카노

  // 실시간 데이터
  currentTemp: number;
  currentWeather: string; // "흐림"
  currentAQI: number;

  // 태그
  tags: string[];         // ["트렌디", "카페많음", "창업가"]

  // 메타
  createdAt: Date;
  updatedAt: Date;
}
```

**Review (리뷰)**

```typescript
interface Review {
```

```typescript
interface Meetup {
```
